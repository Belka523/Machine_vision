# imgproc

**imgproc** — это  консольная утилита и библиотека на языке C для обработки изображений. Для загрузки и сохранения изображений используются популярные заголовочные библиотеки `stb_image` и `stb_image_write`.

Поддерживаемые форматы файлов: **PNG, JPG, BMP, TGA**. Формат сохранения определяется автоматически по расширению выходного файла.

## Возможности

Утилита поддерживает следующие базовые алгоритмы компьютерного зрения:

*   **Медианный фильтр** — эффективно удаляет импульсный шум, сохраняя резкие границы.
*   **Гауссов фильтр** — классическое размытие изображения с настраиваемым размером ядра и параметром сигма.
*   **Детекция границ (Оператор Собеля)** — выделяет контуры объектов на изображении с возможностью установки порога бинаризации.
*   **Матричные свёртки** — применение базовых ядер (повышение резкости, рельеф, обычное размытие).
*   **Коррекция яркости и контраста** — тонкая настройка яркости и контраста изображения.

## Сборка проекта

В проекте есть готовый `Makefile` для сборки с помощью GCC/MinGW.

1. Склонируйте репозиторий или скачайте исходный код.
2. Убедитесь, что установлены заголовочные файлы `stb_image.h` и `stb_image_write.h` .
3. Откройте терминал в папке с проектом и выполните команду:

```bash
mingw32-make
```

В результате будет скомпилирован исполняемый файл `imgproc.exe`.

Для очистки скомпилированных объектных файлов выполните:
```bash
mingw32-make clean
```

## Использование

Общий синтаксис вызова утилиты:

```bash
./imgproc <входной_файл> <операция> [параметры] <выходной_файл>
```

### Доступные операции и параметры:

*   `-median <radius>` — медианный фильтр. Радиус определяет размер окна.
*   `-gauss <size> [sigma]` — гауссов фильтр. `size` — размер ядра (3, 5, 7 и т.д.), `sigma` — сила размытия (0.5 - 3.0).
*   `-edges [threshold]` — оператор Собеля. `threshold`
*   `-convolve <preset>` — применение встроенных матриц свёртки. Доступные пресеты: `sharpen` (резкость), `emboss` (рельеф), `blur` (размытие).
*   `-bc <brightness> <contrast>` — настройка яркости (-255..255) и контраста (0.5..2.0).

### Примеры:

Утилита автоматически конвертирует форматы:

```bash
# Применить медианный фильтр с радиусом 2
./imgproc photo.jpg -median 2 out.png

# Размыть по Гауссу (ядро 5, сигма 1.4)
./imgproc photo.png -gauss 5 1.4 blur.bmp

# Выделить контуры с порогом 60
./imgproc photo.bmp -edges 60 edges.jpg

# Повысить резкость
./imgproc photo.jpg -convolve sharpen sharp.png

# Увеличить яркость на 30 и поднять контраст в 1.5 раза
./imgproc photo.png -bc 30 1.5 final.png
```

## Тестирование

Для автоматической проверки работоспособности всех функций предусмотрен скрипт `test.bat`. 

1. Поместите любое изображение в папку с проектом и назовите его `test.jpg`.
2. Запустите файл `test.bat`.
3. Скрипт запустит все основные фильтры и сохранит результаты в автоматически созданную папку `test_results`.

## Структура проекта

*   `main.c` — консольный интерфейс (CLI) утилиты.
*   `imgproc.h` / `imgproc.c` — ядро библиотеки с реализацией всех алгоритмов.
*   `Makefile` — скрипт автоматической сборки.
*   `test.bat` — скрипт для прогона автоматических тестов.

## Как работают алгоритмы

*   **Медианный фильтр**: Алгоритм проходит по каждому пикселю изображения, собирая его соседей в пределах заданного радиуса. Значения яркости пикселей в этом окне сортируются по возрастанию, и центральному пикселю присваивается медианное значение. Это позволяет эффективно удалять шум, не размывая при этом резкие края объектов.
*   **Гауссов фильтр**: Использует операцию свёртки с ядром, значения которого вычисляются по формуле двумерного распределения Гаусса. В отличие от обычного размытия, Гауссов фильтр придает больший вес пикселям в центре окна и меньший — по краям. Перед применением матрица нормализуется. В результате получается очень мягкое и математически точное сглаживание.
*   **Детекция границ (Оператор Собеля)**: Сначала алгоритм переводит изображение в оттенки серого, используя стандартизированные веса каналов. Затем он вычисляет градиент яркости по осям X и Y, применяя два ядра свёртки 3x3. Итоговая сила границы вычисляется как корень из суммы квадратов обоих градиентов: `sqrt(Gx² + Gy²)`. При задании порога пиксели, чье значение выше него, становятся белыми (граница), а остальные — черными (фон).
*   **Матричные свёртки (Convolution)**: Базовая математическая операция обработки изображений. Небольшая матрица скользит по всему изображению. Новое значение каждого пикселя вычисляется как сумма произведений значений соседних пикселей на соответствующие веса в ядре.
*   **Коррекция яркости и контраста**: Алгоритм применяет линейное преобразование к каждому цветовому каналу пикселя: `новое = контраст * (старое - 128) + 128 + яркость`. Смещение на 128 нужно для того, чтобы изменение контраста происходило относительно "серединного" серого цвета, а не абсолютного черного. Итоговое значение принудительно ограничивается диапазоном `[0..255]`.
